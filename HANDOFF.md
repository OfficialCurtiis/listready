## What it is
ListReady — a free, in-browser image/video tool. Two audiences:
1. **Online sellers** (Etsy/eBay/Amazon/Poshmark): convert HEIC→JPG, resize to marketplace presets, compress, strip GPS.
2. **Anyone with iPhone files on a non-Apple device**: open HEIC on Windows, convert iPhone HEVC `.mov` videos → MP4.

Everything runs **client-side in the browser** (nothing uploaded) — that privacy angle is the core selling point.

## How deploys work (IMPORTANT)
- Edit files locally → **commit to GitHub via the web uploader** (drag files into github.com/OfficialCurtiis/listready/upload/main → Commit) → Vercel auto-builds in ~30–60s.
- **The HTML pages are GENERATED by `build.py`.** To change anything shared (header, nav, footer, the converter UI, FAQ template, or a specific page's title/H1/FAQ), edit `build.py` and run `python3 build.py`, which regenerates index/etsy/ebay/amazon/poshmark/windows .html + sitemap.xml + robots.txt. **Do NOT hand-edit those generated HTML files** — your change gets overwritten next build. `video.html`, `style.css`, `app.js`, `video.js` are standalone (not generated).

## Key technical decisions / gotchas (so we don't repeat mistakes)
- **HEVC video needs the FULL `@ffmpeg/core@0.12.6` (ESM build), loaded from CDN (~30 MB).** The minimal "ffwasm524" HEVC core does NOT include the `mov`/`mp4` demuxer, so it can't read iPhone `.mov` — don't use it.
- The FFmpeg 0.12 wrapper spawns a **module worker**, so the core must be the **ESM** build (it `import()`s it). The worker file (`814.ffmpeg.js`) must be **same-origin** (self-hosted) or browsers block it.
- **4K is the memory killer**, made worse on phones (low memory). `video.js` auto-downscales anything over 1080p (long side → 1920px) before encoding to keep it within browser memory. Verified working.
- **Etsy DOES accept HEIC** now — never claim it doesn't (savvy sellers will call it out). The real Etsy pain is blurry-under-2000px + desktop HEIC upload failures.
- The video `<video>` element won't decode in the automation browser (0×0) — verify output MP4s by parsing bytes (ftyp/moov/avc1), not the preview element.
